# GitHub Publish Checklist

## Before Publishing
1. Review `README.md`
2. Review `docs/release-notes-v0.1.0.md`
3. Review `docs/launch-announcement.md`
4. Confirm `.env.example` matches the current app behavior
5. Confirm `LICENSE`, `CONTRIBUTING.md`, and `SECURITY.md` are present
6. Run `pytest -q`
7. Run the app locally and capture screenshots

## Suggested Screenshots
- homepage hero and status panel
- ATT&CK catalog + ransomware patterns
- incidents + explainable summary
- live ransomware feed panels
- local AI copilot output

## Suggested GitHub Release Assets
- release title: `ATIRF v0.1.0 Public Preview`
- release notes from `docs/release-notes-v0.1.0.md`
- screenshots
- short architecture image if available
