# ATIRF Public Launch Announcement

## Short Version
Today I am publicly releasing **ATIRF - Adaptive Threat Intelligence & Response Framework**.

ATIRF is an open cyber defense platform built to help defenders turn raw telemetry into evidence-backed incident reasoning.

The goal is to combine:

- telemetry ingestion
- CTI enrichment
- MITRE ATT&CK mapping
- attack-pattern correlation
- grounded AI assistance

into a practical workflow for analysts, hunters, researchers, and defender communities.

This project is being built in the open because cyber defense works better when defenders can inspect the logic, validate the mappings, improve the detections, and contribute together.

## What ATIRF Is
ATIRF is not trying to be another generic dashboard.

It is being built as:

- an open detection and incident reasoning platform
- a CTI-aware workflow for defenders
- a community-extensible system for ATT&CK mapping, ransomware tracking, and attack-pattern analysis
- a local-first AI-assisted experience using open models through Ollama where possible

## What The Current Release Includes
- event ingestion and normalization
- enrichment and scoring
- ATT&CK technique catalog seeding
- ransomware playbook matching
- ranked attack-pattern hypotheses
- optional live ransomware tracking via `ransomware.live`
- optional local AI copilot through Ollama
- user-defined CTI connector inventory

## Why Build This In Public
Cyber defenders need tools they can inspect and trust.

The purpose of ATIRF is to create something the community can help shape together:

- better detections
- better ATT&CK mappings
- better CTI connectors
- better public datasets
- better workflows for real investigations

If the community contributes, ATIRF can become more than a demo. It can become a useful open cyber defense platform.

## How Defenders Can Contribute
- submit detections and ATT&CK mappings
- improve ransomware playbook coverage
- add connectors for MISP, OpenCTI, TAXII, KEV, and telemetry providers
- contribute replay datasets and adversary scenarios
- test the platform and report issues
- improve investigation workflows and documentation

## Call To Action
If you are a defender, detection engineer, threat hunter, CTI analyst, researcher, student, or builder in cyber:

- review the project
- break it
- improve it
- contribute to it

The goal is to build something useful together for the broader cyber defense community.
